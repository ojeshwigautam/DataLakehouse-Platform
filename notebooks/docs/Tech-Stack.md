# Technology Stack

## Overview

The Unified Commerce Lakehouse project is built using a modern cloud-native Data Engineering stack that closely resembles technologies used in enterprise environments.

Each technology has been selected based on its relevance in the current Data Engineering job market and its role in building scalable, production-ready data platforms.

---

# Programming Language

## Python

Purpose:
- ETL development
- Data ingestion
- Automation
- Data transformation
- Utility scripts

Reason for Selection:
Python is the industry-standard programming language for Data Engineering because of its rich ecosystem and integration with modern data tools.

---

# Query Language

## SQL

Purpose:
- Data querying
- Business analytics
- Data validation
- Reporting

Reason for Selection:
SQL remains the most essential skill for Data Engineers and is required in almost every data-related role.

---

# Data Processing

## Apache Spark (PySpark)

Purpose:
- Distributed data processing
- Large-scale transformations
- ETL pipelines

Reason for Selection:
Apache Spark is widely adopted for processing large datasets efficiently in enterprise environments.

---

# Workflow Orchestration

## Apache Airflow

Purpose:
- Workflow scheduling
- Pipeline orchestration
- Job dependency management

Reason for Selection:

---

# Cloud Storage

## Amazon S3

Purpose:
- Data Lake Storage
- Bronze Layer
- Silver Layer
